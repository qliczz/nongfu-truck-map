# 电动重卡充电物流地图

浙江省与上海市区域的在线交互地图，标注生产基地与兼容电动重型半挂牵引车（13.75米拖车）的充电站，支持重型货车路径规划（含限行规避）、ETC费用计算、分时段充电费用估算与实时车辆追踪。

## 功能概览

### 地图标注
- **物流大本营** — 调度中心标记，脉冲动画
- **生产基地** — 6个生产基地（建德/淳安），水源地标注
- **重卡充电站** — 21个高速服务区/港口充电站（浙江17个 + 上海4个）
- **小型充电站** — 38个汽车充电站（国网/特斯拉/特来电/星星充电/中国石化等12家运营商）
- **收费站** — 658条收费站数据（浙江11市 + 上海）
- **服务区** — 路径沿途自动识别

### 路径规划
- 任意点到点导航，支持高速优先/国道优先/最短距离三种模式
- **货车限行规避** — 集成高德货车导航API，自动规避限行/禁行区域
- 货车参数：49吨/6轴/18.1m/2.55m宽/4.0m高
- 多途经点支持，路线沿途收费站与服务区自动标注

### 费用计算
- **ETC高速费** — 高德实时API计算（含95折优惠）
- **分时段充电费** — 15分钟粒度分段计算，精确捕捉峰谷切换点
  - 五时段电价：尖峰(2.05x) / 高峰(1.85x) / 平段(1.0x) / 低谷(0.4x) / 深谷(0.2x)
  - 分时站点：电费随时段浮动，服务费固定
  - 统一价站点：全天单一价格
  - 跨时段充电自动分段计费并合并显示
- **充电站真实电价** — 38个站点均配置运营商实际电价（来源：充电桩聚合平台）
- **柴油对比** — 电动重卡 vs 柴油重卡费用对比
- **车辆参数自定义** — 电池容量、能耗、充电功率、费率等全开放编辑

### 车辆追踪
- 集成云查车API，实时显示车辆位置
- 支持云端代理部署，多用户共享
- 代理服务器环境变量配置，无需硬编码凭据

### 其他
- 电价参数本地持久化（localStorage）
- 缩放自适应标记/字体大小
- PC侧边栏 / 手机抽屉式双端兼容
- 后台管理界面（密码保护，支持数据内联编辑与JSON导入导出）

## 快速开始

### 本地运行

```bash
# 克隆仓库
git clone https://github.com/qliczz/charging-logistics-map.git
cd charging-logistics-map

# 启动本地服务器
python -m http.server 8080

# 浏览器访问
# http://localhost:8080
```

### 配置高德API Key

路径规划与坐标验证需要高德地图API Key。请前往[高德开放平台](https://lbs.amap.com/)申请：

```javascript
// js/routing.js 第16行
AMAP_DEFAULT_KEY: 'YOUR_AMAP_KEY',  // 替换为你的Key
```

### 车辆追踪代理（可选）

```bash
# 设置环境变量
export PORT=3001
export VEHICLE_ACCOUNT=your_account
export VEHICLE_PASSWORD=your_password
export VEHICLE_PLATES=plate1,plate2

# 启动代理
node vehicle-proxy.js
```

前端在应用内通过设置面板配置代理URL。

## 分时段电价系统

充电费用计算支持分时段电价（Time-of-Use），根据充电时间跨度的峰谷波动精确计费：

### 计算逻辑

```
分时站点实际电价 = (综合电价 - 服务费) × 时段倍率 + 服务费
统一价站点实际电价 = 综合电价（全天不变）
```

### 时段倍率

| 时段 | 倍率 | 说明 |
|------|------|------|
| 尖峰 | 2.05x | 夏冬季高峰时段 |
| 高峰 | 1.85x | 日间高峰 |
| 平段 | 1.0x | 正常时段 |
| 低谷 | 0.4x | 夜间低谷 |
| 深谷 | 0.2x | 深夜深谷 |

### 季节区分

- **夏冬季**（1/7/8/12月）：含尖峰时段
- **春秋季**（其余月份）：无尖峰时段
- **充电站专属低谷**：每日10:30-14:00

### 跨时段计费示例

17:30开始充电150分钟（国网分时站点 ¥1.39/度）：
- 高峰段 17:30-18:00（30分钟）：¥2.23/度
- 尖峰段 18:00-20:00（120分钟）：¥2.43/度
- 合计费用按各段电量分别计算后求和
